# Frontend Elite — Design & Implementation Guide

> This skill is the **supreme law** for all frontend work. No exceptions. No shortcuts.
> The goal: interfaces that feel designed by a world-class team — not generated by an AI
> defaulting to purple gradients, Inter font, and desktop-only layouts.

---

## PHASE 0 — Design Thinking Before Any Code

**Never write code before answering these five questions internally.**
Poor upfront decisions cannot be fixed later without a full rewrite.

### The Five Pre-Code Questions

**1. Who is the user and what context are they in?**
- Developer scanning dense info on a large monitor
- Consumer making an emotional decision on their phone
- Executive reviewing a dashboard on a tablet
- Creative professional on a laptop with extended display
- Casual user on an older mid-range Android phone

The user context determines density, font size floors, touch targets, and animation intensity.

**2. What is the primary device context?**

| Context | Implications |
|---------|-------------|
| Mobile-first consumer app | 44px+ touch targets, thumb-zone navigation, bottom sheets, simplified nav |
| Desktop productivity/SaaS | Dense information, hover states, keyboard shortcuts, sidebar navigation |
| Tablet editorial/media | Touch + pointer hybrid, landscape/portrait adaptation, generous margins |
| Responsive marketing site | Fluid typography, stacking layouts, hero sections that work at all widths |
| Dashboard (all devices) | Breakpoint-specific data density, collapsible sidebar, responsive charts |

**3. What is the single aesthetic direction?**

Pick exactly ONE. Execute it with full commitment. Timid half-measures produce forgettable output.

| Direction | Feeling | Visual Signature |
|-----------|---------|------------------|
| `luxury-dark` | Expensive, timeless, silent | Near-black, gold/cream accent, wide tracking, serif display |
| `editorial-magazine` | Authoritative, readable, cultured | Paper-warm bg, ink-black text, red accent, strong serifs |
| `industrial-raw` | Direct, functional, honest | Dark concrete, orange/lime accent, condensed or monospace |
| `futuristic-cold` | Precise, advanced, cutting-edge | Deep navy/black, cyan/neon accent, geometric type, grid lines |
| `organic-warm` | Natural, approachable, trustworthy | Cream/sand, terracotta/forest accent, humanist serif |
| `playful-kinetic` | Energetic, fun, expressive | Warm light bg, bold coral accent, rounded type, micro-motion |
| `brutalist-raw` | Uncompromising, bold, confrontational | Pure white, black 2px borders, one harsh accent |
| `zen-minimal` | Calm, focused, deliberate | Off-white, near-invisible borders, ink accent, vast whitespace |
| `data-dense` | Analytical, trustworthy, precise | Neutral dark, blue/teal accent, tabular mono, tight spacing |
| `retro-nostalgic` | Familiar, warm, character-rich | Aged yellows/greens, slab serif, grainy textures |

**4. What is the ONE thing someone remembers after seeing this?**

If you cannot answer this in one sentence, the design will be forgettable. Every decision serves this.

**5. Dark or light? Why?**
- Dark: dev tools, entertainment, luxury, gaming
- Light: editorial, e-commerce, health, B2B enterprise
- **Never default to the same choice twice.** Actively vary it.

## PHASE 1 — Color System

### ❌ ABSOLUTELY FORBIDDEN

These are the fingerprints of AI-generated design. Any appearing in output is a failure:

- `#6366f1` / `#8b5cf6` / `#a78bfa` / `#7c3aed` as primary accent on white
- `linear-gradient(135deg, #667eea, #764ba2)` — the most overused hero bg in existence
- Teal (`#14b8a6`) + white + Inter — the canonical 2021 default SaaS aesthetic
- `#6b7280` gray as the only secondary color
- Glassmorphism `backdrop-blur` on gradient backgrounds — overused since 2022
- 4+ competing accent colors in one component
- Gradient rounded buttons on white cards

### ✅ Required: Full Token Architecture

```css
/* ─── MASTER COLOR TOKEN SYSTEM ─────────────────────────────────── */
:root {
  /* Surfaces */
  --color-bg:             #0a0a0a;
  --color-bg-elevated:    #111111;
  --color-surface-1:      #161616;
  --color-surface-2:      #1e1e1e;
  --color-surface-3:      #262626;

  /* Borders */
  --color-border:         #2a2a2a;
  --color-border-strong:  #3a3a3a;
  --color-border-subtle:  #1e1e1e;

  /* Text — minimum 3 levels */
  --color-text-primary:   #f0ece4;
  --color-text-secondary: #9a9591;
  --color-text-muted:     #5a5650;
  --color-text-inverse:   #0a0a0a;

  /* Accent — ONE primary + states */
  --color-accent:         #d4a843;
  --color-accent-hover:   #e8c05a;
  --color-accent-active:  #b88c2e;
  --color-accent-subtle:  rgba(212, 168, 67, 0.12);

  --color-success:        #4caf7d;
  --color-success-subtle: rgba(76, 175, 125, 0.12);
  --color-warning:        #f0a842;
  --color-warning-subtle: rgba(240, 168, 66, 0.12);

  --color-info:           #4a9edd;
```

**60/30/10 rule:** 60% surfaces, 30% text+borders, 10% accent.

## PHASE 2 — Typography

### ❌ ABSOLUTELY FORBIDDEN

These fonts are the single biggest indicator of AI-generated output. Zero exceptions:

- **Inter** — as display/heading font. Acceptable for dense data tables only.
- **Roboto** — no personality whatsoever, do not use
- **Space Grotesk** — massively overused in AI UIs since 2022, immediately reads as AI slop
- **Poppins** — overused to the point of meaninglessness in consumer products
- **Nunito Sans** — weak, forgettable
- **Montserrat** — overused since 2018, completely generic now
- **Arial / Helvetica / system-ui** — never as a deliberate brand choice
- **Source Sans Pro / Open Sans** — the "I didn't pick a font" fonts

### ✅ Font Selection — Distinctive & Character-Driven

**→ See `references/typography-pairs.md` for full pairings, loading code, and CSS.**

## PHASE 3 — Icons: SVG Rules

### Core Rule

**All UI icons must be SVG.** Emojis are content, never interface elements.

### When Emojis ARE Acceptable (narrow exceptions)

1. Explicitly casual/social/celebratory context (confetti screen, chat bubble)
2. User explicitly requests them
3. `playful-kinetic` direction where emoji ARE the aesthetic
4. Emoji lives in content copy, not as an interactive element

**Never use emoji as:** nav items, action buttons, status badges, form icons, tab bar icons.

### Framework-Specific Strategy

#### React / Next.js

**Lucide React** — default choice for most projects
```bash
npm install lucide-react
```
```jsx
import { Search, Home, ArrowRight, X, ChevronDown } from 'lucide-react'

// Always set both size and strokeWidth
<Search size={20} strokeWidth={1.5} className="text-[var(--color-text-muted)]" />

// In a button
<button className="flex items-center gap-2">
  Continue <ArrowRight size={16} strokeWidth={2} />
</button>
```

**Phosphor** — when you need weight variants (thin → duotone)
```bash
npm install @phosphor-icons/react
```
```jsx
import { House, Gear } from "@phosphor-icons/react"
<House size={24} weight="duotone" />  // thin/light/regular/bold/fill/duotone
```

**Heroicons** — best for Tailwind-only projects
```bash
npm install @heroicons/react
```
```jsx
import { MagnifyingGlassIcon } from '@heroicons/react/24/outline'  // or /24/solid
<MagnifyingGlassIcon className="w-5 h-5" />
```

> Pick ONE library per project. Stroke styles differ subtly and mixing looks inconsistent.

#### Vue / Nuxt
```bash
npm install -D unplugin-icons @iconify/json
# vite.config.ts → Icons({ compiler: 'vue3' })
```
```html
<script setup>
import IconSearch from '~icons/lucide/search'
</script>
<IconSearch class="w-5 h-5" />
```

#### Svelte / SvelteKit
```bash
npm install unplugin-icons  # same approach
# or: npm install svelte-feather-icons
```

#### Plain HTML — Inline SVG

```html
<svg xmlns="http://www.w3.org/2000/svg"
     width="20" height="20" viewBox="0 0 24 24"
     fill="none" stroke="currentColor"
     stroke-width="1.5" stroke-linecap="round" stroke-linejoin="round"
     aria-hidden="true" focusable="false">
  <!-- PATH HERE -->
</svg>
```

### SVG Rules — Non-Negotiable

1. **`viewBox` is mandatory** — always `viewBox="0 0 24 24"` for 24px grid icons
2. **`currentColor`** — always `stroke="currentColor"` or `fill="currentColor"`, never hardcoded hex
3. **Accessibility** — decorative icons: `aria-hidden="true"`. Icon-only buttons: `aria-label` on the button
4. **Consistent stroke-width** — choose 1.5 (refined) or 2 (standard) and never mix within one project
5. **Mobile touch targets** — the SVG itself can be 20px but its clickable wrapper must be ≥44×44px

```css
/* Icon button touch target — correct approach */
.icon-btn {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  min-width: 44px;   /* touch target */
  min-height: 44px;
  padding: 10px;     /* centers a 20px icon in 44px space */
  border-radius: 8px;
  cursor: pointer;
}
/* The SVG inside is 20px but the clickable area is 44px — never make icons tiny tap targets */
```

## PHASE 4 — Responsive Design System

> **This is not optional.** Every frontend output must work beautifully at all screen sizes.
> "I'll make it responsive later" = it never gets done.

### Breakpoint Strategy

```css
/* ─── BREAKPOINT TOKENS ──────────────────────────────────────────── */
:root {
  --bp-xs:  320px;   /* small phones (SE, Galaxy A series) */
  --bp-sm:  480px;   /* large phones */
  --bp-md:  768px;   /* tablets portrait */
  --bp-lg:  1024px;  /* tablets landscape, small laptops */
  --bp-xl:  1280px;  /* desktops */
  --bp-2xl: 1536px;  /* large desktops, wide monitors */
  --bp-3xl: 1920px;  /* full HD and above */
}
```

**Always mobile-first:** write base styles for mobile, then `min-width` overrides upward.

```css
/* ✅ Mobile-first — correct */
.component { font-size: 1rem; }
@media (min-width: 768px)  { .component { font-size: 1.125rem; } }
@media (min-width: 1024px) { .component { font-size: 1.25rem; } }

/* ❌ Desktop-first — avoid */
.component { font-size: 1.25rem; }
@media (max-width: 768px) { .component { font-size: 1rem; } }
```

### Container System

### Navigation Patterns — Device-Appropriate

**The most critical responsive component. Must be designed for all three states:**

```html
<!-- Navigation structure that adapts to all devices -->
<nav class="nav" role="navigation" aria-label="Main navigation">
  <!-- Desktop: horizontal top bar | Mobile: hamburger + drawer -->
  <div class="nav-inner container">
    <a class="nav-logo" href="/">Logo</a>

    <!-- Desktop links — hidden on mobile -->
    <ul class="nav-links" role="list">
      <li><a href="/about">About</a></li>
      <li><a href="/work">Work</a></li>
      <li><a href="/contact">Contact</a></li>
    </ul>

    <!-- CTA — visible on desktop, hidden on mobile (or shown in drawer) -->
    <a class="btn btn-primary nav-cta" href="/start">Get Started</a>

    <!-- Hamburger — only visible on mobile -->
    <button class="nav-toggle" aria-label="Open menu" aria-expanded="false" aria-controls="nav-drawer">
```

## PHASE 8 — Performance Optimization

> Performance IS design. A beautiful interface that loads slowly or janks on scroll is a failed
> product. Every output must target: LCP < 2.5s, CLS < 0.1, INP < 200ms.

**→ For deep patterns per framework, see `references/performance.md`.**

### The Four Performance Metrics (Core Web Vitals)

| Metric | What It Measures | Good | Needs Work | Poor |
|--------|-----------------|------|------------|------|
| **LCP** Largest Contentful Paint | When the main content loads | < 2.5s | 2.5–4s | > 4s |
| **CLS** Cumulative Layout Shift | Visual stability (content jumping) | < 0.1 | 0.1–0.25 | > 0.25 |
| **INP** Interaction to Next Paint | Responsiveness to user input | < 200ms | 200–500ms | > 500ms |
| **FCP** First Contentful Paint | When first content appears | < 1.8s | 1.8–3s | > 3s |

### Critical Rule: Never Ship These Performance Killers

```
❌ Images without width + height attributes (causes CLS)
❌ Fonts without display=swap (blocks render)
❌ @import in CSS (serial network requests)
❌ render-blocking <script> without defer/async
❌ Uncompressed images > 200KB for hero, > 80KB for thumbnails
❌ Importing entire icon libraries (import * from 'lucide-react')
❌ Loading third-party scripts in <head> without defer
❌ CSS transitions on properties that trigger layout (width, height, top, left, margin)
❌ Animations on non-composited properties (use transform + opacity only)
❌ No lazy loading on below-the-fold images
❌ fetch() calls inside useEffect with no abort controller (memory leaks + race conditions)
```

### Images — The Biggest LCP Win

**Always use the correct format and attributes:**

```html
<!-- ✅ Hero image — highest priority, never lazy-load the LCP element -->
<img
  src="hero.webp"
  alt="Description"
  width="1200"
  height="630"
  fetchpriority="high"
  decoding="async"
/>

<!-- ✅ Below-fold image — lazy load, prevent layout shift -->
<img
  src="feature.webp"
  alt="Description"
  width="800"
  height="600"
  loading="lazy"
  decoding="async"
/>

<!-- ✅ Responsive image with multiple sizes -->
<img
  srcset="img-400.webp 400w, img-800.webp 800w, img-1200.webp 1200w"
  sizes="(max-width: 640px) 100vw, (max-width: 1024px) 50vw, 800px"
  src="img-800.webp"
  alt="Description"
  width="800"
  height="600"
  loading="lazy"
  decoding="async"
/>

<!-- ✅ Art direction: different crop for mobile vs desktop -->
<picture>
  <source media="(max-width: 639px)"  srcset="img-portrait.webp"  width="640"  height="853">
  <source media="(min-width: 640px)"  srcset="img-landscape.webp" width="1200" height="630">
  <img src="img-landscape.webp" alt="Description" width="1200" height="630" loading="lazy">
</picture>
```

**Format selection:**
- `WebP` — default for all photos and complex images (30–50% smaller than JPEG)
- `AVIF` — even better compression than WebP, Chrome 85+/Firefox 93+ (use with WebP fallback)
- `SVG` — logos, icons, illustrations with flat colors
- `PNG` — only when transparency + sharp edges needed (screenshots, product UI)
- Never use `GIF` — use `<video autoplay loop muted playsinline>` instead

```html
<!-- Animated content: video instead of GIF — 10–50x smaller -->
<video autoplay loop muted playsinline width="640" height="360">
  <source src="animation.webm" type="video/webm">
  <source src="animation.mp4"  type="video/mp4">
</video>
```

### CSS Performance — Composited Properties Only

**Only animate these two properties for 60fps on every device:**

```css
/* ✅ COMPOSITED — runs on GPU, never causes layout/paint */
.animate-good {
  transition: transform 250ms ease, opacity 250ms ease;
}
.animate-good:hover {
  transform: translateY(-4px) scale(1.01);
  opacity: 0.9;
}

/* ❌ LAYOUT-TRIGGERING — causes full reflow, janky on mobile */
.animate-bad {
  transition: width 250ms, height 250ms, top 250ms, margin 250ms;
}
```

**Promote elements to their own compositor layer (use judiciously):**
```css
/* Pre-promote elements that will animate, preventing repaint of surrounding content */
.will-animate {
  will-change: transform;  /* Only set this on elements ABOUT to animate */
  /* Remove it after animation completes — don't apply globally */
}

/* For static elements that overlap animated ones */
```

**Avoid layout thrashing in JS:**
```js
// ❌ Layout thrash — reads then writes in a loop
elements.forEach(el => {
  const height = el.offsetHeight;  // READ — forces layout
  el.style.marginTop = height + 'px';  // WRITE — invalidates layout
});

// ✅ Batch reads, then batch writes
const heights = elements.map(el => el.offsetHeight);  // batch READ
elements.forEach((el, i) => el.style.marginTop = heights[i] + 'px');  // batch WRITE

// ✅ Even better — use requestAnimationFrame
requestAnimationFrame(() => {
  const heights = elements.map(el => el.offsetHeight);
  requestAnimationFrame(() => {
    elements.forEach((el, i) => el.style.marginTop = heights[i] + 'px');
  });
});
```

### Lazy Loading & Intersection Observer

```js
// ✅ Native lazy loading for images (use this first)
<img loading="lazy" src="..." alt="..." width="..." height="...">

// ✅ Intersection Observer for custom lazy patterns
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.src = entry.target.dataset.src
      observer.unobserve(entry.target)  // stop observing after load
    }
  })
}, {
  rootMargin: '200px',  // start loading 200px before visible
  threshold: 0
})

document.querySelectorAll('[data-src]').forEach(img => observer.observe(img))
```

**React — lazy load below-fold sections:**
```jsx
import { useState, useEffect, useRef } from 'react'

function LazySection({ children }) {
  const ref = useRef(null)
  const [visible, setVisible] = useState(false)

  useEffect(() => {
    const observer = new IntersectionObserver(([entry]) => {
      if (entry.isIntersecting) {
        setVisible(true)
        observer.disconnect()
      }
    }, { rootMargin: '300px' })

    if (ref.current) observer.observe(ref.current)
    return () => observer.disconnect()
  }, [])

  return (
    <div ref={ref}>
      {visible ? children : <div style={{ minHeight: '400px' }} />}
    </div>
  )
}
```

### Caching Strategy

```
Static assets (images, fonts, JS, CSS):
  Cache-Control: public, max-age=31536000, immutable
  (1 year — use content hash in filename for cache busting)

HTML pages:
  Cache-Control: public, max-age=0, must-revalidate
  (always revalidate — or use CDN edge caching)

API responses:
  Cache-Control: private, max-age=60  (short, user-specific data)
  Cache-Control: public, max-age=300  (5 min, public data)
  Cache-Control: no-store             (sensitive data)
```

**Service Worker for offline + instant repeat visits:**
```js
// Basic cache-first strategy for static assets
self.addEventListener('install', event => {
  event.waitUntil(
    caches.open('v1').then(cache => cache.addAll([
      '/', '/styles/main.css', '/js/app.js', '/fonts/display.woff2'
    ]))
  )
})

self.addEventListener('fetch', event => {
  event.respondWith(

    )
  )
})
```

### INP — Interaction Responsiveness

**Keep the main thread unblocked:**
```js
// ❌ Synchronous heavy computation blocks input response
function processLargeArray(arr) {
  return arr.map(item => expensiveOperation(item)) // freezes UI
}

// ✅ Yield to browser between chunks
async function processLargeArrayAsync(arr) {
  const results = []
  for (let i = 0; i < arr.length; i++) {
    results.push(expensiveOperation(arr[i]))
    if (i % 50 === 0) {
      await new Promise(resolve => setTimeout(resolve, 0)) // yield
    }
  }
  return results
}

// ✅ Web Worker for CPU-intensive tasks off main thread
const worker = new Worker('/workers/heavy-computation.js')
worker.postMessage({ data: largeArray })
worker.onmessage = ({ data }) => setResult(data)
```

**Debounce and throttle expensive handlers:**
```js
// Search input — debounce prevents firing on every keystroke
const handleSearch = debounce(async (query) => {
  const results = await api.search(query)
  setResults(results)
}, 300)

// Scroll handler — throttle to max 1 per frame
const handleScroll = throttle(() => {
  setScrollY(window.scrollY)
}, 16) // ~60fps

// Resize — debounce, expensive to respond to every pixel
const handleResize = debounce(() => {
  recalculateLayout()
}, 150)
```
